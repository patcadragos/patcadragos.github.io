Original — Python & Pygame

Where it started

Tank Duel started as a desktop game written in Python using the Pygame library. The project explored 2D game architecture — scrolling worlds wider than the screen, camera tracking, AI opponents, and a physics-based bullet arc that applies gravity only after a certain travel distance.

The original codebase handled asset loading, sprite scaling, collision detection, power-up spawning, and multi-level progression — all within a classic Pygame event loop. Music and SFX were layered on top using pygame.mixer.

  • 3 levels with increasing AI aggression
  • 1-player (vs AI) and 2-player local co-op modes
  • 3 power-up types: HEALTH, SPEED, DAMAGE
  • Scrolling 1600×600 world with camera following Player 1
  • Bullets arc downward past 400px travel — skill-based long shots

Rewriting for the browser

The goal was a faithful port — same logic, same constants, same AI behaviour — but running natively in the browser using the Canvas 2D API and Web Audio. No game engine, no framework. Three excerpts below show the most interesting translation challenges.

01 — Fixed-timestep game loop / TankBattle.html 
// Pygame's clock.tick(FPS) is replaced with an accumulator loop.
// This keeps physics deterministic regardless of frame rate.
const MS_PER_FRAME = 1000 / FPS; // FPS = 60

function gameLoop(ts) {
  if (!running) return;

  if (lastT === null) lastT = ts;
  accum += ts - lastT;
  lastT = ts;

  // Run one logic tick per 16.67 ms accumulated
  while (accum >= MS_PER_FRAME) {
    accum -= MS_PER_FRAME;
    if (!tick()) { running = false; break; }
  }

  render();
  if (running) requestAnimationFrame(gameLoop);
}

requestAnimationFrame(gameLoop);

Pygame's blocking clock.tick(60) becomes a requestAnimationFrame loop with an accumulator. Physics runs at a locked 60 Hz step; rendering runs as fast as the display allows.

02 — Enemy AI (handle_ai) / direct port from Python 
// Mirrors handle_ai() from the Python version exactly:
// random direction change every 20-60 frames, random fire chance.
function handle_ai() {
  if (ai_state.timer <= 0) {
    // randint(-1, 1): pick left, idle, or right
    ai_state.direction = [-1, 0, 1][Math.floor(Math.random() * 3)];
    ai_state.timer = 20 + Math.floor(Math.random() * 41); // randint(20, 60)
  }

  yellow.x += ai_state.direction * lvl.aiSpeed; // lvl.aiSpeed scales per level
  yellow.x = Math.max(0, Math.min(yellow.x, WORLD_W - TANK_W));
  ai_state.timer -= 1;

  // ~1/51 chance to fire each frame (random.randint(0,50)==0)
  if (Math.floor(Math.random() * 51) === 0 && shoot_cooldown.yellow === 0) {
    const dx = yellow_facing === 1 ? BULLET_SPEED : -BULLET_SPEED;
    const bx = dx > 0 ? yellow.x + TANK_W : yellow.x - BULLET_W;
    bullets.push({ owner: 'yellow', x: bx, y: yellow.y,
                   w: BULLET_W, h: BULLET_H, dx, dy: 0, travel: 0 });
    shoot_cooldown.yellow = COOLDOWN;
  }
}

The AI is deliberately simple — random walk with random shooting — but level 3 uses aiSpeed: 4 and pwFreq: 200 to make it feel harder without changing the core logic.

03 — Bullet arc & collision / physics mechanic 
const BULLET_GRAVITY_START = 400; // px of travel before drop kicks in
const GRAVITY = 0.4;              // dy added per frame after threshold

bullets = bullets.filter(b => {
  b.travel += Math.abs(b.dx); // track horizontal distance covered

  // Only apply gravity past 400px — short shots fly flat
  if (b.travel > BULLET_GRAVITY_START) b.dy += GRAVITY;

  b.x += b.dx;
  b.y += b.dy;

  // Out of bounds — remove
  if (b.y > SH || b.x < 0 || b.x > WORLD_W) return false;

  // AABB hit detection
  function rectCollide(a, b) {
    return a.x < b.x + b.w && a.x + a.w > b.x
        && a.y < b.y + b.h && a.y + a.h > b.y;
  }

  if (b.owner === 'yellow' && rectCollide(b, red)) {
    health.red -= 10 * effects.yellow.damage; // damage powerup multiplier
    return false;
  }
  if (b.owner === 'red' && rectCollide(b, yellow)) {
    health.yellow -= 10 * effects.red.damage;
    return false;
  }

  return true; // bullet survives this frame
});

Bullets travel flat for the first 400 px, then arc downward — rewarding long-range skill shots while keeping close combat snappy. AABB collision keeps it O(n) and accurate at the game's small bullet sizes.
